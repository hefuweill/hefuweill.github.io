Java 虚拟机规范学习笔记

前言

由于最近在学使用 Transform 结合 ASM 进行字节码插桩，而 ASM 是直接操作字节码的，而我对字节码及具体字节码指令不是很清楚，于是就买了这本书先看看，记录一下方便复习。

运行时数据区

Java 虚拟机定义了以下几种程序运行期间会使用到的运行时数据区。

pc 寄存器

pc 寄存器为线程私有的，每个线程都有一个。如果当前执行的为 Native 方法，那么 pc 寄存器的值为 undefined，如果当前执行的不是 Native 方法，那么 pc 寄存器的值为当前正在执行字节码指令的地址。

Java 虚拟机栈

Java 虚拟机栈为线程私有的，每个线程都有一个。这个栈与线程同时创建，用于存储栈帧，而栈帧中主要用于存储数据和部分过程结果。

栈帧

栈帧是用于存储数据和部分过程结果的数据结构，同时也用于处理动态链接、方法返回值和异常分派。

栈帧随着方法调用而创建，随着方法结束而销毁（包括正常完成以及异步完成）。

栈帧存储空间由创建它的线程分配在 Java 虚拟机栈中，每一个栈帧都有一个本地变量表、一个操作数栈以及一个指向当前方法所在类常量池的引用 。其中本地变量表和操作数栈的容量在编译期确定（因此如果使用 ASM 操作字节码需要考虑容量是否变化）。

局部变量表

每个栈帧内部都包含一个局部变量表，其长度在编译期决定，保存在方法的 code 属性上。

一个局部变量可以保存一个类型为 boolean、byte、char、short、int、float、reference 或 returnAddress （指向某个操作码的指针）的数据，两个局部变量可以保存一个类型为 long 或者 double 的数据。

局部变量通过索引来进行定位访问，首个局部变量的索引值取值范围为大于等于 0，小于局部变量表的长度，long 和 double 类型的数据值采用两个局部变量中较小的索引值来定位。

Java 虚拟机使用局部变量表来完成方法调用时的参数传递。当调用类方法时，它的参数将会被依次传递到局部变量表中从 0 开始的连续位置上，当调用类方法时，第 0 个局部变量一定用于存储该对象所在对象的引用（也就是 this），后续的其它参数将会传递到局部变量表中从 1 开始的连续位置上。

操作数栈

每个栈帧内部都包含一个操作数栈，其最大深度在编译期决定，保存在方法的 code 属性上。

在栈帧刚刚创建时，操作数栈是空的。Java 虚拟机提供一些字节码指令来从局部变量表或者对象实例的字段中复制常量或变量值到操作数栈中，也提供一些指令用于从操作数栈中取走数据、操作数据以及把操作结果重新入栈。在调用方法时，操作数栈也用来准备调用方法的参数以及方法返回结果。

操作数栈的每个位置上可以保存一个 Java 虚拟机中定义的任意数据类型的值，包括 long 和 double。在任意时刻操作数栈都会有一个确定的栈深度，一个 long 或者 double 类型的数据占用两个单位栈深度，其他数据类型占用一个单位栈深度。[栈深度等于栈元素个数加 long、double 类型元素数量？]

动态链接

每个栈帧都包含一个指向当前方法所在类常量池的引用 ，以便对当前方法的代码实现动态链接。在 class 文件中一个方法若要调用其它方法，或者访问成员变量，都需要通过符号引用表示。动态链接的作用就是将符号引用替换为运行时的真实引用。

类加载过程中会解析尚未被解析的符号引用，并且将对变量的访问转换为变量在程序运行时，位于存储结构中的正确偏移量。

方法调用正常完成

方法调用正常完成是指执行过程过程中没有向上抛出异常，在这种情况下当前栈帧承担着恢复调用者状态的责任，包括恢复调用者的局部变量表以及操作数栈[为什么要恢复这两个？调用方法时也没更改这两。]以及正确递增程序计数器，以跳过刚刚执行的方法指令。调用者的代码会在被调用方法返回值压入操作数栈后继续执行。

方法调用异常完成

方法调用异常完成是指执行过程过程中向上抛出了异常，在这种情况下一定有返回值返回给其调用者。

Java 堆

Java 堆为线程共享的，其在虚拟机启动的时候就被创建，是供所有类实例和数组类对象分配内存的区域。

方法区

方法区为线程共享的，其在虚拟机启动的时候就被创建，它存储了每个类的结构信息，例如运行时常量池、字段和方法数据、构造方法和普通方法的字节码内容，还包括一些在类实例初始化时用到的特殊方法。方法区不一定存储在 Java 堆中。

特殊方法

对于虚拟机而言，构造器是以一个名为 的特殊实例初始化方法的形式出现的。 这个方法是由编译器命名的，因为它不是一个有效的标识符，无法通过编码得到。该方法只能在实例初始化期间使用虚拟机指令 invokespecial 来调用，并且只能在未初始化的实例上调用，构造器的访问权限会约束其衍生出的 方法。

每个类都含有类初始化方法 ，其不包含参数，返回值为 void，名称也是由虚拟器命名的，因为它不是一个有效的标识符，无法通过编码得到。类就是通过该方法进行初始化的，注意该方法只能由虚拟机自身隐式调用,没有任何虚拟机指令可以主动调用它。

运行时常量池

每一个运行时常量池都在方法区中分配，其是 class 文件中每一个类或接口的常量池表的运行时表示形式，在加载类或接口到虚拟机后，就会创建对应的运行时常量池。注：class 文件中常量池存储符号引用，运行时常量池存储直接引用。

本地方法栈

本地方法栈是线程私有的，每个线程都有一个，该栈用于支持 native 方法的调用。

异常

Java 虚拟机中异常的出现总是由下面三种原因之一导致的。

  1. athrow 字节码指令被执行。

  2. 虚拟机同步检测到程序发生了非正常的执行情况。

    a：字节码指令蕴含的语义违反了 Java 语言的语义，比如访问一个超出数组边界索引的元素。

    b：程序在加载或者连接过程中发生错误。

  3. 由于以下原因导致异步异常的出现。

    a：调用了 Thread 或者 ThreadGroup 的 stop 方法。

    b：Java 虚拟机实现发生了错误。

每个方法都会有零至多个异常处理器，异常处理器描述了其在方法代码中的有效作用范围，能处理的异常类型以及处理异常的代码所在位置，当方法抛出异常后虚拟机会按顺序搜索当前方法所有异常处理器，如有某个能处理就跳转到对应代码执行，都不能处理，那么销毁当前栈帧，返回上个栈帧并重新抛出，依次类推，如果已经到达调用链顶端还没有找到能处理该异常的异常处理器则终止当前线程。

字节码指令

Java 虚拟机的指令有一个字节长度的（最多 128 种）、代表着某种特定操作含义的操作码以及其后的零至多个操作数所构成。许多指令并不包含操作数只有一个操作码。如果忽略异常处理，那么 Java 虚拟机的解释器通过下面的这个伪代码的循环即可有效工作。
