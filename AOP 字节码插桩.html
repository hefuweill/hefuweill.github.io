前言

由于现在公司业务都是使用 H5 进行开发，而原生也就接入了一个 Bugly ，但它只能检测崩溃、ANR，无法得知哪些网页加载缓慢、哪些资源加载错误，也就没办法进行针对性优化。因此打算收集 WebView 相关回调帮助发现问题。借助 Android Plugin 中的 Transform 结合 ASM 进行无侵入埋点。

Transform

什么是 Transform？

android 构建打包插件从 2.0 版本开始，加入了 Transform API，它允许我们在 class 转换为 dex 前做一些处理。借助于此就能实现无痕埋点。

基本使用

首先需要自定义一个 Gradle 插件，重写 apply 方法注册 Transform。

方法内部就是先判断插件列表中是否有 AppPlugin（com.android.application） 这个插件，如果有那么获取 AppExtension 实例，注册 Transform。

  • getName 获取 Transform 名称
  • getInputTypes 获取需要处理的内容 CLASSES、RESOURCES。
  • getScopes 获取需要处理的范围 PROJECT、SUB_PROJECT、EXTERNAL_LIBRARIES、TESTED_CODE、PROVIDED_ONLY。
  • isIncremental 获取是否支持增量编译。

其实每注册一个 Transform 都会生成一个 Task，名称为 transform{inputType}With{name}For{variant}。在编译完成后就会执行该 Task 也就是调用 transform 方法，所以重点看看该方法。

方法参数为 TransformInvocation，看看其内部都有些什么

一般而言不需要关心 getReferencedInputs 以及 getSecondaryInputs 这两个方法，只需关心下面三个方法。

  • getContext 获取 Transform 的上下文，可用于获取项目名称，Task 、Variant 名称等。
  • getInputs 获取 Transform 的输入源，需要修改输入源然后复制到目标路径做为下一个 Transform 的输入源。
  • getOutputProvider 获取输出提供者，可用于获取每个输入对应的目标地址。
